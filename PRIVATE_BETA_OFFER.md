# Offre bêta privée

MINI_MCP_CLIENT est actuellement proposé uniquement en bêta privée et démonstration encadrée.

Le code source complet n’est pas distribué publiquement pour le moment.

## Ce qui peut être proposé

### 1. Démonstration privée

Objectif :

- présenter le couloir local MCP ;
- montrer le daemon local ;
- expliquer l’inventaire des outils ;
- discuter d’un cas d’usage concret.

Format :

- courte session guidée ;
- pas de publication du code complet ;
- pas d’accès aux secrets ou données privées.

### 2. Diagnostic bêta privé — 50 $ CAD

Objectif :

- évaluer si MINI_MCP_CLIENT peut correspondre à votre contexte ;
- identifier les outils locaux utiles ;
- repérer les limites et risques ;
- proposer une première orientation technique.

Livrable possible :

- court retour écrit ;
- recommandations prudentes ;
- liste des scénarios compatibles ou non compatibles.

### 3. Bêta guidée limitée — 150 $ CAD

Objectif :

- accompagner une exploration plus concrète ;
- préparer une démonstration adaptée ;
- vérifier un workflow ou un couloir de diagnostic ;
- expliquer comment un LLM pourrait passer par une couche MCP locale.

Ce format reste limité et encadré. Il ne correspond pas à une livraison complète du code source.

## Public cible

Cette bêta peut intéresser :

- développeurs Windows ;
- utilisateurs de LLM locaux ou distants ;
- consultants IA ;
- petites équipes techniques ;
- builders MCP ;
- personnes qui veulent tester agents IA, fichiers, terminal, SQLite, workflows, logs et tests.

## Ce qui n’est pas inclus

L’offre bêta n’inclut pas :

- distribution publique du code complet ;
- licence open source ;
- garantie de production ;
- sécurité absolue ;
- promesse d’autonomie totale ;
- remplacement de Codex, Claude, Gemini ou Qwen ;
- développement sur mesure illimité.

## Positionnement honnête

MINI_MCP_CLIENT est un runtime local expérimental en validation terrain.

La valeur proposée au départ est :

- démonstration ;
- diagnostic ;
- évaluation technique ;
- accompagnement limité ;
- discussion de faisabilité.

## Demander une démonstration

Vous pouvez demander une démonstration privée ou une évaluation technique en décrivant :

- votre environnement Windows ;
- les LLM que vous utilisez ;
- les outils locaux que vous voulez tester ;
- le niveau de confidentialité requis ;
- le type de workflow ou diagnostic souhaité.
